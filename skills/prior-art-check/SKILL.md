"Novel to me" is not "novel to the field." Before investing in a direction, find the closest existing work — not to be discouraged, but to position the contribution honestly and to avoid reinventing a solved problem. Finding out it's been done is a cheap result now and an expensive one in six months.

## Workflow
- **Search from multiple entry points** → keywords, the canonical papers' citation graph (forward and backward), key authors, and adjacent fields with the same structure. One database and one query is not a search.
- **Identify the closest prior work** → the 3–5 papers that most overlap; for each, state what they did and exactly where they stop.
- **Locate the gap precisely** → is the contribution a new regime, method, dataset, scale, or a contradiction of a prior claim?
- **Classify novelty honestly** → genuinely new / incremental extension / replication / already done. Each is a valid finding; name it.
- **Record the search** → queries, databases, date, and closest hits, so the related-work claim is defensible and re-runnable.

## Anti-patterns
- Declaring novelty after one keyword search in one database.
- Confusing "no one cites these together" with "no one has done it."
- Stopping at the first confirming or contradicting paper.
- Citing only supportive work and missing the paper that already did it.

## Hard vs adaptable
- **Hard rule:** a novelty claim is backed by a recorded search of more than one route, with the closest prior work named.
- **Adaptable:** search breadth scales to the commitment — a quick check before a side experiment, an exhaustive sweep before a thesis.
